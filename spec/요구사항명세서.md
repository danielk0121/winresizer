# 개발 요구사항 명세서 (SRS): 개인 맞춤형 macOS 윈도우 매니저

## 1. 프로젝트 개요
* **목표:** macOS 환경에서 전역 단축키를 이용해 활성화된 창의 위치와 크기를 제어하는 개인용 백그라운드 유틸리티 개발.
* **타겟 사용자:** 1인 (개발자 본인). 복잡한 GUI 설정 화면이나 앱스토어 배포 과정을 배제하고, 로컬 환경에서 즉각적인 코드 수정과 반영이 가능한 실용성을 최우선으로 함.

## 2. 시스템 환경 및 기술 스택
* **운영체제:** macOS
* **개발 언어:** Python 3.x
* **핵심 라이브러리:**
    * `pynput`: OS 전역 키보드 이벤트 후킹 (Global Hotkey 캡처).
    * `pyobjc`: macOS 네이티브 API (Accessibility API, AppKit) 접근 및 윈도우 제어.

## 3. 기능적 요구사항 (Functional Requirements)
* **REQ-F01: 전역 단축키 감지**
    * 다른 앱을 사용 중인 상태에서도 지정된 키 조합(예: `Option + Command + 방향키`)이 입력되는 것을 백그라운드에서 상시 감지해야 함.
* **REQ-F02: 활성 윈도우 식별**
    * 단축키 입력 시점에 최상단(Frontmost)에 활성화된 애플리케이션을 찾고, 해당 윈도우의 접근성 객체 핸들(`AXUIElement`)을 획득해야 함.
* **REQ-F03: 듀얼 모니터 환경 인식 및 좌표 매핑**
    * 사용 중인 27인치 메인 모니터와 24인치 서브 모니터의 해상도 및 물리적 배치에 따른 전체 작업 영역 좌표를 기준으로 계산함.
    * 윈도우 크기 계산 시 상단 메뉴바와 하단 Dock 영역을 침범하지 않는 실제 사용 가능 영역(`visibleFrame`)을 구해야 함.
* **REQ-F04: 윈도우 재배치 및 크기 조절**
    * 입력된 단축키에 맵핑된 동작에 따라 타겟 윈도우의 위치(`kAXPositionAttribute`)와 크기(`kAXSizeAttribute`) 값을 덮어씀.
    * **예상 동작 예시:** * 좌/우 절반 분할
        * 27인치 모니터의 정확한 중앙에 특정 해상도(예: 1200x800)로 띄우기
        * 활성 창을 24인치 서브 모니터로 즉시 넘기기

## 4. 비기능적 요구사항 (Non-Functional Requirements)
* **REQ-N01: 백그라운드 실행 (Daemonize)**
    * 터미널 세션 종료와 무관하게 동작해야 하며, Mac 부팅 및 로그인 시 자동으로 실행되도록 `launchd` (plist 작성) 또는 간단한 `nohup` 방식을 활용함.
* **REQ-N02: 시스템 권한 확보**
    * 앱 정상 동작을 위해 macOS '시스템 설정 > 개인정보 보호 및 보안'에서 스크립트를 실행하는 환경(터미널 등)에 다음 두 가지 권한을 부여해야 함.
        1.  **손쉬운 사용 (Accessibility):** 다른 앱의 윈도우 속성 제어 권한.
        2.  **입력 모니터링 (Input Monitoring):** 백그라운드 키보드 입력 감지 권한.
* **REQ-N03: 극단적인 유지보수성 (Customization)**
    * 화면 분할 비율이나 단축키 매핑 규칙은 별도의 설정 파일이나 UI 없이, Python 스크립트 상단의 딕셔너리나 변수에 하드코딩하여 관리함. (개발자 본인만 사용하므로 직관적인 코드 수정이 가장 빠름)

## 5. 단계별 구현 및 테스트 계획 (Milestones)
* **Phase 1 (이벤트 캡처 PoC):** `pynput`만 사용하여 백그라운드에서 `Opt + Cmd + Left` 입력 시 콘솔에 텍스트가 출력되는지 확인.
* **Phase 2 (윈도우 제어 PoC):** `pyobjc`만 사용하여 현재 활성화된 터미널 창의 너비를 강제로 반으로 줄이는 단일 스크립트 테스트 (이때 접근성 권한 부여 필요).
* **Phase 3 (기능 통합):** Phase 1의 단축키 콜백 함수 내부에 Phase 2의 윈도우 제어 로직을 결합.
* **Phase 4 (최적화 및 정착):** 27인치와 24인치 모니터 환경에 맞춘 세부 좌표값 하드코딩, 예외 처리(크기 조절이 불가능한 창 등) 추가 후 `launchd`에 백그라운드 서비스로 등록.
