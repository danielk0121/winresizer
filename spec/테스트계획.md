# 테스트 계획 및 전략

## 1. 개요
TDD(Test-Driven Development) 원칙을 준수하여, 기능 구현 전 테스트 코드를 먼저 작성하고 이를 통과시키는 방식으로 개발을 진행한다.

## 2. 테스트 환경
- **언어:** Python 3.x
- **프레임워크:** `unittest` 또는 `pytest`
- **핵심 테스트 대상:**
    - 단축키 감지 로직
    - 활성 윈도우 식별 및 핸들 획득
    - 모니터 해상도에 따른 좌표 계산 로직
    - 윈도우 위치/크기 변경 명령 실행 여부

## 3. 테스트 케이스 설계 (TDD 사이클)
### 1단계: 단위 테스트 (Unit Tests)
- **모니터 유틸리티:** 각 모니터의 `visibleFrame` 값을 정확히 가져오는지 확인.
- **좌표 계산기:** 입력된 비율(예: 좌측 50%)에 따른 정확한 (x, y, w, h) 좌표값 반환 검증.

### 2단계: 통합 테스트 (Integration Tests)
- **단축키 콜백:** 특정 키 입력 시 해당 명령이 정상적으로 트리거되는지 확인.
- **윈도우 제어 API:** `pyobjc`를 통해 실제 윈도우 속성이 변경되는지(Mocking 또는 실제 환경 테스트) 확인.

## 4. 자동화 및 검증
- 모든 커밋 전 테스트 스위트를 실행하여 회귀 테스트를 수행한다.
