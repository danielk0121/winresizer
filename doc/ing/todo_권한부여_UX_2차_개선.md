# 작업 목록: 권한 부여 UX 2차 개선 (입력 모니터링 및 재실행 안내)

## 개요
현재 '손쉬운 사용' 권한 위주로 설계된 UX를 '입력 모니터링' 권한까지 확장하고, macOS 보안 정책상 권한 부여 후 반드시 필요한 '앱 재실행' 과정을 사용자가 자연스럽게 수행하도록 개선한다.

## 현재 문제점
1. **입력 모니터링 권한 누락**: 단축키 감지에 필수적인 '입력 모니터링' 권한 상태를 체크하지 않음. (해결됨)
2. **재실행 안내 부족**: 권한 허용 후 앱을 재실행해야만 실제 기능이 작동함에도 불구하고, 사용자가 웹 UI 새로고침만 반복하는 현상 발생. (해결됨)
3. **단계별 시각화 부재**: 여러 권한 중 어떤 것이 부족한지 직관적으로 알기 어려움. (해결됨)

## 할 일 목록

### 1. 백엔드 기능 확장
- [x] `is_input_monitoring_trusted()` 로직 구현 (`Quartz.CGPreflightListenEventAccess` 활용).
- [x] `/api/status` 응답에 `input_monitoring_granted` 필드 추가.

### 2. 웹 UI/UX 고도화
- [x] **가이드 팝업 4단계 구성**:
    1. 손쉬운 사용 (접근성) 설정
    2. 입력 모니터링 설정
    3. 앱 재실행 (필수) 안내
    4. 완료 안내
- [x] **실시간 상태 시각화**: 각 단계를 완료할 때마다 체크표시(취소선/흐림 처리)로 진행 상황 표시.
- [x] **강력한 종료/재실행 안내**: 모든 권한 설정 완료 시 "앱을 다시 실행해 주세요"라는 강조된 메시지 제공.

### 3. 기술적 보완
- [ ] 권한 획득 여부를 감지하여 시스템 알림(Notification)으로 "설정 완료! 앱을 다시 실행해 주세요" 메시지 발송. (추후 진행 가능)

## 향후 시나리오
1. 사용자: "설정 열기" 클릭 -> 손쉬운 사용 허용.
2. 웹 UI: 1단계 취소선 처리 -> 2단계 "입력 모니터링 설정 열기" 활성화.
3. 사용자: 2단계 허용.
4. 웹 UI: "모든 설정 완료! 앱을 다시 실행하세요" 안내 노출.
5. 사용자: 앱 종료 후 재실행 -> 단축키 정상 작동 확인.
