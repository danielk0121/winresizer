# 작업 목록: 설치 및 권한 부여 UX 편의성 개선

## 개요
WinResizer 설치 후 사용자가 '손쉬운 사용' 권한을 부여하는 과정에서 겪는 혼란을 최소화하고, macOS 보안 정책(권한 변경 시 앱 종료)에 대응하는 매끄러운 사용자 경험을 설계한다.

## 할 일 목록

### 1. 웹 UI 안내 강화
- [ ] **초기 진입 안내**: 권한이 없을 경우, 설정 페이지 중앙에 "WinResizer 시작하기" 가이드 팝업 또는 섹션 노출.
- [ ] **단계별 가이드 시각화**: 
    1. [클릭] 시스템 설정 열기
    2. [체크] WinResizer 허용
    3. [재실행] 앱 다시 열기 (종료됨을 미리 안내)
- [ ] **권한 변경 후 안내**: "권한을 변경하면 앱이 자동으로 종료될 수 있습니다. 종료 후에는 트레이 아이콘을 통해 다시 실행해 주세요." 문구 추가.

### 2. 백엔드 및 앱 동작 개선
- [ ] **설정창 호출 버그 수정**: 현재 작동하지 않는 웹 UI 배지의 "설정 창 열기" 링크(`open_accessibility`) 동작 보완.
- [ ] **최초 실행 시 자동 안내**: 앱이 처음 실행되었을 때 권한이 없다면, 사용자에게 알림(Notification)을 보내 설정을 유도.

### 3. 문서 및 도움말 보강
- [ ] **FAQ 추가**: "왜 권한을 줬는데 앱이 꺼지나요?" 등 빈번한 질문에 대한 답변 준비.
- [ ] **README 업데이트**: 설치 및 권한 부여 과정을 스크린샷과 함께 상세히 설명.

## 예상 UX 시나리오
1. 사용자가 앱 실행 → 웹 설정 페이지 자동 오픈.
2. 상단에 빨간색 "권한 필요" 배지 노출.
3. 배지 클릭 시 설정 창 열림 + 웹 페이지에 "설정 후 앱이 종료되면 다시 실행해 주세요" 메시지 표시.
4. 사용자가 설정 완료 → 앱 종료.
5. 사용자가 앱 재실행 → 초록색 "권한 승인됨" 배지 확인 및 단축키 사용 시작.
