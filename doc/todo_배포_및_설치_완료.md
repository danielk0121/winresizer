# 배포 및 설치 계획서

## 1. 개요
WinResizer를 macOS 환경에서 일반 사용자가 편리하게 설치하고 실행할 수 있도록 독립 실행형 애플리케이션 번들(`.app`) 및 디스크 이미지(`.dmg`) 형태로 패키징하는 계획을 기술합니다.

**보안 이슈 (필수 반영 사유):**
현재처럼 터미널(또는 iTerm2)에서 Python 스크립트 형태로 실행할 경우, macOS의 '손쉬운 사용(Accessibility)' 권한을 터미널 전체에 부여해야 합니다. 이는 터미널에서 실행되는 모든 프로세스에 시스템 제어 및 키로거 권한을 허용하게 되는 심각한 보안 취약점(Over-privileging)을 발생시킵니다. 따라서 최소 권한의 원칙(Principle of Least Privilege)을 준수하기 위해 반드시 터미널과 분리된 독립적인 `.app` 번들로 배포되어야 합니다.

## 2. 패키징 및 배포 상세
### 2.1 독립 실행형 앱 번들링
- **목표:** 터미널이나 Python 설치 없이 실행 가능한 `.app` 파일 생성 및 **터미널과 접근성 권한 분리**
- **도구:** `PyInstaller` 또는 `py2app` 활용
- **주요 작업:**
    - 의존성 라이브러리(PyQt5, pynput, pyobjc) 포함
    - 앱 아이콘 및 메타데이터(Info.plist) 설정
    - **접근성 권한을 앱 자체(`WinResizer.app`)에만 독립적으로 부여할 수 있도록 서명 및 구성**

### 2.2 DMG 디스크 이미지 생성
- **목표:** 표준 macOS 설치 경험(드래그 앤 드롭) 제공
- **도구:** `create-dmg` 활용
- **구성:**
    - `/Applications` 폴더 바로가기 포함
    - 배경 이미지 및 아이콘 배치 커스터마이징

### 2.3 앱 아이콘 및 비주얼
- **목표:** macOS 스타일의 전용 앱 아이콘 적용
- **포맷:** `.icns` 파일 (다양한 해상도 대응)

## 4. TDD 검증 계획 (테스트 시나리오)

**※ OS 레벨(시스템) E2E 검증의 필요성:**

일반적인 Python 단위 테스트와 달리, 배포용 애플리케이션(`.app`)과 접근성 권한 분리 과정은 철저히 **macOS 운영체제와의 상호작용**에 의존합니다. 

파이썬 가상 환경에서 스크립트를 실행하는 것과, 컴파일된 바이너리가 OS로부터 `손쉬운 사용` 권한을 어떻게 요청하고 처리하는지는 완전히 다른 동작 메커니즘을 가지기 때문입니다. 

따라서 실제 사용자 환경과 100% 동일하게 앱을 설치/실행하고 시스템 권한이 정상 분리되었는지(보안 이슈 해결 여부)를 검증하기 위해서는, 

테스트 스크립트가 직접 `open` 명령어나 `hdiutil` 명령어로 OS 자원을 제어하고 상태를 확인하는 **운영체제 레벨의 E2E 테스트가 필수적**입니다.

패키징 및 배포 프로세스의 신뢰성과 독립 앱으로서의 권한 분리 동작을 검증하기 위한 시나리오입니다. 

빌드 파이프라인 테스트와 E2E 동작 검증을 병행합니다. (자동화 스크립트와 수작업 검증 혼합)

**[검증 주체 및 방법 분리]**
- **AI(자동화 스크립트) 영역:** 빌드(`PyInstaller` 등) 스크립트 실행, 생성물(`.app`, `.dmg`)의 디렉토리 및 파일 구조 검사, 시스템 명령어를 통한 앱 실행 및 DMG 마운트/언마운트 과정까지 테스트 코드로 100% 자동화합니다.
- **사용자(수작업) 개입 영역:** macOS의 강력한 보안 정책으로 인해 접근성 권한 팝업 클릭이나 우회는 OS에서 원천 차단됩니다. 따라서 앱이 실행될 때 "손쉬운 사용 권한 허용" 알림창이 **'터미널'이 아닌 'WinResizer' 앱 자체**를 가리키는지 확인하고 권한 스위치를 켜는 보안 검증의 최종 확인은 사용자의 육안과 손으로 직접 진행해야 합니다.

1. **앱 번들 빌드 검증 (`test_app_bundle_creation`)**
   - 빌드 스크립트(`PyInstaller` 등) 실행 후 `dist/WinResizer.app` 디렉토리가 정상적으로 생성되는지 자동화 스크립트로 확인합니다.
   - 필수 내부 구조(`Contents/MacOS/`, `Contents/Resources/icon.icns`, `Info.plist`)가 올바르게 구성되어 있는지 검증합니다.

2. **접근성 권한 독립성 검증 (`test_accessibility_permission_isolation`)**
   - 기존의 파이썬 인터프리터(터미널) 방식이 아닌, 빌드된 `WinResizer.app`을 직접 실행(`open -a WinResizer.app`)합니다.
   - **(핵심 보안 검증)** 앱 실행 시 macOS 시스템 환경설정에서 터미널이 아닌 **'WinResizer' 앱 자체**에 대한 '손쉬운 사용' 권한을 새롭게 요구하는지 확인합니다.
   - 권한이 없는 상태에서 실행 시 정상적으로 경고 대화상자(안내)가 표시되는지 E2E로 확인합니다.

3. **DMG 생성 및 구조 검증 (`test_dmg_creation_and_mount`)**
   - DMG 생성 스크립트 실행 후 최종 배포용 `.dmg` 파일이 성공적으로 생성되는지 검증합니다.
   - 생성된 DMG를 마운트(`hdiutil attach`)하여 내부에 `WinResizer.app`과 `/Applications` 심볼릭 링크가 정상적으로 포함되어 있는지 확인합니다.

## 버그 리포트
- dmg 파일로 설치를 한 후 실행하면, 앱이 실행이 안된다. 트레이 아이콘도 생기지 않는다.
- venv 로 파이썬을 직접 실행하면 잘 실행되는 상태다.
- 우선, dmg 파일로 설치 후 실행했을때, 로그 파일이 어디에 남는지 확인이 필요하다.
- 지금, 로그를 확인할 수가 없다. 

## 설치 파일 고용량 이슈
- dmg 파일 용량 160MB
- dmg 파일이 앱으로 설치된 이후 용량 약 400MB
- 파이썬 프로그램 약 3MB
- 용량 줄이기가 필요하다. 파이썬 GUI 프레임워크 때문에 용량이 큰 것으로 판단된다.
