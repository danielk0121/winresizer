# 버그 리포트: 요구사항에 없는 '사이클' 기능 노출

## 1. 현상 설명
- '좌측 절반' 또는 '우측 절반' 단축키를 반복해서 입력할 경우, 창의 크기가 1/2 -> 1/3 -> 2/3 -> 1/2 순으로 순환(Cycle)하며 변경됨.
- 이는 요구사항 명세서(`spec/요구사항명세서.md`)에 정의되지 않은 기능이며, 사용자의 의도와 다르게 창 크기가 변할 수 있음.

## 2. 재현 경로
1. 임의의 창을 활성화함.
2. `⌥⌘←` (좌측 절반) 단축키 입력 -> 창이 좌측 절반으로 이동함.
3. 다시 `⌥⌘←` 입력 -> 창이 좌측 1/3로 변경됨 (버그).
4. 다시 `⌥⌘←` 입력 -> 창이 좌측 2/3로 변경됨 (버그).
5. 다시 `⌥⌘←` 입력 -> 창이 다시 좌측 절반으로 돌아옴 (버그).

## 3. 원인 분석
- `app/src/gui.py`의 `execute_window_command` 함수 내에 현재 창의 크기를 체크하여 다음 모드로 강제 전환하는 '스마트 순환' 로직이 포함되어 있음.
- 해당 로직은 요구사항 검토 없이 구현된 것으로, 사용자가 설정한 단축키의 본래 목적(항상 좌측 절반 배치)을 저해함.

## 4. 해결 방안
- `app/src/gui.py`에서 `is_nearly_equal` 함수와 `execute_window_command` 내의 조건부 모드 전환 로직(if/elif)을 삭제함.
- 단축키 입력 시 해당 모드(`좌측_절반`, `우측_절반` 등)의 좌표로만 이동하도록 단순화함.
