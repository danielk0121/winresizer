# 멀티모니터 창 이동 및 사이클링 기능 개선

## 개요
- 창 크기/위치 변경 단축키를 반복 입력할 때, 현재 모니터의 경계에 도달하면 다음 모니터로 창을 이동시키고 상태를 관리한다.
- 단순히 위치만 반복하는 것이 아니라, 모니터 간의 자연스러운 전환(Restore -> Position)을 지원한다.

## 상세 요구사항 (1/2 우측 정렬 예시)
1. **현재 상태**: 좌측 모니터에 창이 있음.
2. **1회 입력**: 좌측 모니터의 **우측 1/2** 영역으로 정렬.
3. **2회 입력**: 더 이상 우측으로 이동할 공간이 없으므로, **우측 모니터**로 이동하며 **Restore(원래 크기 또는 기본 위치)** 상태로 변경.
4. **3회 입력**: 우측 모니터에서 **우측 1/2** 영역으로 정렬.
5. **4회 입력**: 더 이상 우측으로 이동할 공간이 없으므로, 다시 **좌측 모니터(사이클링)**로 이동하며 **Restore** 상태로 변경.

## [버그 수정 완료] 작은 창 크기(최소 크기 제한 앱) 이동 불가 이슈
### 현상
- 사용자가 설정한 비율(예: 35%)이 앱(크롬 등)의 최소 폭 제한보다 작을 경우, 앱이 더 이상 줄어들지 않아 `is_similar` 판단(x, y, w, h 모두 비교)을 벗어남.
- 이로 인해 "이미 정렬된 상태"로 인식되지 못하고 다음 모니터로 넘어가지 않는 버그 발생.

### 수정 근거 (거짓 없는 실증적 해결)
1. **단순 오차 허용(Tolerance)의 한계 인지**: 이전 수정에서 `tolerance=50`을 적용했으나, 브라우저의 최소 폭은 실제 계산값과 100~200px 이상 차이 날 수 있음을 확인 (사용자 스크린샷 증거).
2. **엣지 기반 판정(Edge-based Detection) 도입**: 
   - 창의 너비(`w`)나 높이(`h`)가 일치하지 않더라도, 사용자가 요청한 **방향의 끝단(Edge)**이 모니터 경계에 도달해 있다면 정렬 완료로 판단.
   - 예: `Right` 정렬 시 `(창의 x + 너비)`가 `(모니터의 x + 너비)`와 일치(오차 20px 이내)하면 크기 상관없이 다음 모니터로 이동.
3. **실증적 테스트 완료**: 
   - `app/tests/test_multi_monitor_cycling.py`에서 **너비 오차 150px** 상황(계산 400px vs 실제 550px)을 강제로 부여하여 테스트 수행.
   - 개선된 엣지 판정 로직을 통해 해당 상황에서도 정상적으로 모니터 이동이 트리거됨을 검증 완료.

## 구현 계획 및 결과
- [x] `window_controller`에서 모니터 간 창 이동 시 `restore` 상태 적용 기능 구현.
- [x] **[수정]** `is_similar`에 의존하지 않고, 모드별 경계 도달 여부를 판정하는 엣지 기반 로직 추가.
- [x] **[검증]** 크롬 브라우저처럼 폭이 줄어들지 않는 상황(150px 오차)에서도 사이클링 성공 확인.
