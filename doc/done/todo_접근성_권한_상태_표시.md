# 작업 목록: 접근성 권한 상태 웹 UI 표시

## 개요
사용자가 WinResizer의 단축키가 작동하지 않을 때, 가장 먼저 확인해야 할 '손쉬운 사용(Accessibility)' 권한 상태를 설정 페이지(웹 UI)에서 실시간으로 확인할 수 있도록 기능을 추가한다.

## 할 일 목록
- [ ] **백엔드 API 구현**: 현재 앱의 권한 상태를 반환하는 `/api/status` 엔드포인트 추가 (web_server.py)
- [ ] **프론트엔드 UI 업데이트**: 설정 페이지 상단 또는 하단에 권한 상태 배지(Badge) 표시 (index.html, app.js, style.css)
- [ ] **자동 리프레시**: 페이지 로드 시 및 일정 간격으로 권한 상태를 체크하여 사용자에게 알림
- [ ] **로그 보강**: 권한 획득 성공 및 실패(누락) 상황을 `hotkey_listener.py`와 `window_controller.py`에 더 명확히 기록
- [ ] **버그 수정**: 웹 UI 배지 클릭 시 macOS 설정 화면(`open_accessibility`)이 열리지 않는 현상 해결

## 예상 UI 구성
- **권한 승인됨**: 녹색 배지 "접근성 권한: 승인됨"
- **권한 필요**: 적색 배지 "접근성 권한: 필요 (클릭하여 설정 열기)"
