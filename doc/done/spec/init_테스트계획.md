# 테스트 계획 및 전략 (최신본)

## 1. 개요
단축키 감지, 윈도우 조절 로직, 좌표 계산 등 핵심 모듈의 신뢰성을 확보하기 위해 Mock 기반의 단위 테스트와 통합 테스트를 수행한다.

## 2. 테스트 환경
- **프레임워크:** Python 내장 `unittest`
- **실행 명령:** `PYTHONPATH=. app/venv/bin/python3 -m unittest discover app/tests`
- **핵심 테스트 대상:**
    - `CoordinateCalculator`: 다양한 모니터 해상도 및 분할 모드별 좌표 산출 정확성
    - `MonitorInfo`: macOS Native API를 통한 모니터 영역 정보 획득 및 가공
    - `ConfigManager`: JSON 파일의 읽기/쓰기 및 데이터 무결성
    - `WindowManager`: macOS 접근성 API를 이용한 실제 윈도우 제어 로직 (Mock 활용)

## 3. 테스트 케이스 설계
### 3.1 단위 테스트 (Unit Tests)
- **좌표 계산기 테스트:**
    - `1/2`, `1/3`, `2/3` 등 모든 모드에서 화면 영역을 벗어나지 않는지 검증.
    - 정수 연산 오차 방지 확인.
- **모니터 정보 테스트:**
    - 다중 모니터 환경에서 `visibleFrame`의 원점(Origin)과 크기(Size) 정보를 올바르게 파싱하는지 확인.
- **설정 관리자 테스트:**
    - `config.json` 파일 생성 및 데이터 무작위 변경 후 다시 불러왔을 때 일치 여부 확인.

### 3.2 통합 및 시스템 테스트
- **GUI 및 리스너 연동:**
    - 단축키 버튼 클릭 후 키 입력 시 `HOTKEY_CONFIG`가 즉시 갱신되는지 확인.
    - 리스너 재시작 후 새로운 단축키가 실제로 작동하는지(수동 검증 병행) 확인.
- **권한 검증:**
    - 시스템 접근성 권한이 없을 때 경고 메시지가 올바르게 출력되는지 확인.

## 4. 검증 및 배포 조건
- 모든 `unittest` 케이스 통과 필수.
- macOS 12 및 13 버전에서의 실행 확인.
- `config.json`의 유효성 검사 통과.
