# 분석 리포트: 권한 설정 변경 시 프로세스 자동 종료 현상

## 현상
- 사용자가 `시스템 설정 > 개인정보 보호 및 보안 > 손쉬운 사용(또는 입력 모니터링)`에서 WinResizer의 권한을 삭제하거나 다시 추가하는 과정에서 실행 중이던 WinResizer 프로세스가 예기치 않게 종료됨.

## 원인 분석
### 1. macOS 시스템 수준의 강제 종료
- **보안 메커니즘**: macOS는 TCC(보안 권한) 목록에서 앱이 제거되거나 권한이 변경될 때, 보안 무결성을 유지하기 위해 해당 앱의 모든 활성 프로세스를 즉시 종료(SIGKILL)시킨다.
- **의도된 동작**: 이는 WinResizer의 버그가 아니라, macOS 운영체제의 의도된 보안 정책이다. 사용자가 권한을 뺏거나 설정을 초기화할 때 앱이 계속 돌아가며 권한을 오용하는 것을 방지하기 위함이다.

### 2. 라이브러리(pynput/CGEventTap)의 반응
- 단축키 엔진이 사용하는 `CGEventTap`은 권한 상태에 직접적으로 의존한다. 시스템 설정에서 권한이 해제되면 이벤트 탭이 즉시 `Disabled` 상태가 되며, 이 과정에서 프로세스가 종료되지 않더라도 단축키 엔진은 더 이상 작동할 수 없는 상태가 된다.

## 대응 방안
- **사용자 안내**: "권한 설정 변경 시 앱이 안전을 위해 자동으로 종료될 수 있습니다. 설정을 마친 후 앱을 다시 실행해 주세요."라는 안내를 웹 UI 또는 도움말에 추가.
- **자동 재실행 로직(검토)**: 시스템에 의해 종료되는 경우 앱 차원에서 스스로를 재실행하기는 어려우나, 헬퍼 프로세스를 통해 감시하는 방안이 있으나 복잡도가 높아 권장하지 않음.
- **결론**: 현재로서는 **"정상적인 macOS 보안 동작"**으로 판단하며, 설정 변경 후 앱을 재시작하는 것을 표준 가이드로 확정함.
