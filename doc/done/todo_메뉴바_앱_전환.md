# 메뉴바(상태 표시줄) 백그라운드 앱 전환 계획

## 1. 개요
현재 WinResizer는 실행 시 일반적인 창 형태로 동작하고 있으나, 창 관리 유틸리티의 특성상 백그라운드에서 항상 실행되며 필요할 때만 설정 창을 띄우는 방식이 더 적합합니다. 
macOS 우측 상단 시계 옆(메뉴바/상태 표시줄)에 아이콘 형태로 상주하는 백그라운드 앱으로 전환하는 계획을 기술합니다.

## 2. 기술적 가능성 및 구현 방향
**파이썬으로 구현 가능합니다.** 현재 프로젝트에서 이미 사용 중인 프레임워크와 라이브러리를 활용하여 두 가지 주요 접근 방식이 있습니다.

1. **PyQt5 (권장):**
   - 이미 설정 GUI를 위해 `PyQt5`를 사용 중이므로, 추가 라이브러리 설치 없이 `QSystemTrayIcon` 클래스를 사용하여 메뉴바 아이콘과 드롭다운 메뉴를 구현할 수 있습니다.
   - 유지보수가 용이하고 의존성을 늘리지 않아 가장 추천되는 방식입니다.

2. **rumps 라이브러리:**
   - macOS 전용 메뉴바 앱을 만들기 위한 파이썬 라이브러리입니다. 내부적으로 `pyobjc`를 사용하여 매우 네이티브한 느낌을 줍니다. 
   - 메뉴바 기능에 특화되어 있으나, 기존 PyQt5 GUI와 이벤트 루프를 통합해야 하는 추가 작업이 필요할 수 있습니다.

## 3. 주요 요구사항 및 동작 방식
- **백그라운드 실행:** 앱 실행 시 메인 설정 창은 숨김 상태(Hidden)로 백그라운드에서 단축키 감지(리스너)만 동작해야 함.
- **메뉴바 아이콘:** macOS 메뉴바에 WinResizer 전용 아이콘 표시.
- **드롭다운 메뉴 구성:**
  - `설정 (Preferences...)`: 클릭 시 기존에 만든 PyQt5 설정 창 오픈.
  - `종료 (Quit)`: 앱 및 백그라운드 단축키 리스너 완전 종료.
- **창 닫기 동작 변경:** 설정 창에서 '닫기(X)' 버튼을 누르거나 'Quit' 버튼 대신 '닫기'를 누를 경우, 앱이 종료되는 것이 아니라 창만 숨겨지고 메뉴바에서는 계속 실행되도록 수정.

## 5. TDD 검증 계획 (테스트 시나리오)

메뉴바(System Tray) 모드 전환에 따른 GUI 및 프로세스 라이프사이클의 정확성을 TDD 방식으로 검증합니다. 이를 위해 새로운 테스트 파일인 `app/tests/test_tray_e2e.py`를 신규 생성하여 아래 케이스들을 작성합니다.

1. **초기 실행 상태 검증 (`test_initial_hidden_state`)**
   - 앱이 처음 실행될 때 메인 윈도우 인스턴스가 생성되지만 화면에 표시되지 않는지(`isHidden() == True`) 확인합니다.
   - `QSystemTrayIcon` 인스턴스가 정상적으로 생성되고 표시 상태(`isVisible() == True`)인지 확인합니다.

2. **창 닫기(Close Event) 오버라이드 검증 (`test_close_event_hides_window`)**
   - 메인 설정 창이 띄워진 상태(`show()`)에서 창 닫기(Close) 이벤트를 트리거합니다.
   - 창이 완전히 파괴(Destroy)되거나 애플리케이션 프로세스가 종료(`QApplication.quit` 미호출)되지 않고, 단순히 숨김(`isHidden() == True`) 상태로 전환되는지 확인합니다.

3. **트레이 메뉴 동작 검증 (`test_tray_menu_actions`)**
   - **설정 메뉴 클릭:** 트레이 메뉴의 'Preferences...' 액션을 프로그래밍 방식으로 트리거(Trigger)했을 때, 메인 창이 다시 화면에 표시(`isVisible() == True`)되는지 확인합니다.
   - **종료 메뉴 클릭:** 트레이 메뉴의 'Quit' 액션을 트리거했을 때, `QApplication.quit()` 시그널이 정상적으로 발생하고 단축키 리스너 스레드가 안전하게 종료되는지 Mocking을 통해 검증합니다.

4. **단축키 리스너 백그라운드 유지 검증 (`test_listener_alive_when_hidden`)**
   - 메인 창이 숨겨진 상태(`isHidden() == True`)에서도 `HotkeyListenerThread` 인스턴스가 여전히 동작 중(`isRunning() == True`)인지 확인합니다.

5. **단축키로 창 크기 조절 및 이동 검증 (`test_window_resize_and_move_via_hotkey`)**
   - 기존 단위 테스트(`app/tests/test_main.py`)처럼 `execute_window_command` 내부 동작을 모킹(Mocking)하지 않고, 실제 E2E(End-to-End) 방식으로 검증합니다.
   - 트레이 모드(백그라운드) 실행 상태에서 `pynput` 컨트롤러 또는 시스템 레벨의 키보드 이벤트를 실제로 발생시켜 설정된 단축키를 입력합니다.
   - 활성화된 테스트용 창(예: Chrome 또는 텍스트 편집기)의 실제 좌표와 크기(`Bounds`)가 운영체제 레벨에서 예상된 비율과 위치로 변경되었는지 검증합니다.
