# 버그 리포트: 웹 UI 설정 페이지 초기 진입 시 음영 노출 이슈

## 현상
- 웹 설정 페이지(`index.html`)에 처음 진입했을 때, 사용자가 아무런 수정을 하지 않았음에도 불구하고 일부 항목에 노란색 음영(수정됨 표시)이 나타남.
- 특히 커스텀 비율 창 조절 항목과 일부 기본 단축키 항목에서 발생함.

## 원인 추정
1. **커스텀 비율 항목**: 페이지 로드 시 HTML 템플릿의 기본값(75%)과 실제 설정 파일의 값(예: 60%)이 다를 경우, `app.js`가 DOM 값을 업데이트하기 전에 `isModified` 체크가 먼저 수행되어 발생함.
2. **비교 로직 타이밍**: `loadConfigUI` 함수 내에서 `renderHotkeys` 및 `renderCustomHotkeys` 호출 순서와 DOM 업데이트 시점의 불일치.

## 해결 방안
- `loadConfigUI`에서 모든 DOM 요소의 값을 먼저 설정한 후, 음영 처리를 위한 렌더링 함수를 호출하도록 수정.
- `isModified` 함수 내에서 DOM 값을 직접 참조하는 대신, 내부 상태 객체(`config`)와 초기 상태 객체(`initialConfig`)를 비교하도록 개선.
