# 버그 리포트: 시스템 트레이 아이콘 투명 표시 이슈

## 1. 개요
macOS 시스템 트레이(메뉴바)에 앱이 백그라운드로 실행될 때, 드롭다운 메뉴(설정, 종료)는 정상적으로 동작하고 클릭 영역도 존재하지만 정작 아이콘 이미지가 화면에 표시되지 않고 투명하게 보이는 버그가 발생함.

## 2. 현상 (재현 경로)
1. 앱을 실행하여 백그라운드 메뉴바 모드로 진입.
2. macOS 우측 상단의 메뉴바 영역을 확인.
3. 앱이 위치해야 할 자리가 비어있는 것처럼(투명하게) 보이며, 해당 빈 공간을 클릭하면 "설정 (Preferences...)" 및 "종료 (Quit)" 메뉴가 정상적으로 팝업됨.

## 3. 원인 추정
1. **경로 문제 (가장 유력):** `gui.py`에서 아이콘을 로드할 때 상대 경로(`app/src/ui/tray_icon.png`)를 사용하고 있음. 앱을 실행하는 워킹 디렉토리(CWD)나 번들링된 `.app` 내부에서는 해당 상대 경로가 올바르지 않아 이미지를 찾지 못하고 투명한 기본 아이콘(또는 로드 실패)으로 렌더링될 가능성이 큼.
2. **이미지 리소스 형식 문제:** 생성한 `tray_icon.png`가 PyQt5의 `QIcon`에서 제대로 인식되지 않거나, 투명도 채널(Alpha)이 macOS 메뉴바 렌더링 방식과 충돌할 가능성.
3. **macOS 다크/라이트 모드 대비 문제:** 아이콘의 색상이 현재 macOS 테마 색상과 완전히 동일하여 시각적으로 보이지 않는 문제. (하지만 투명하게 공간만 차지하는 것으로 보아 로드 실패 가능성이 더 큼).

## 4. 해결 방안 (수정 계획)
- 아이콘 로드 경로를 현재 스크립트(`gui.py`) 위치를 기준으로 하는 **절대 경로**로 변경하여, 어디서 앱을 실행하든 항상 올바른 아이콘 파일을 찾을 수 있도록 수정.
  - 예: `os.path.join(os.path.dirname(__file__), 'ui', 'tray_icon.png')` 적용.
- PyInstaller로 패키징된 바이너리 환경(`sys._MEIPASS`)과 일반 파이썬 실행 환경을 모두 대응할 수 있도록 경로 탐색 유틸리티 함수 적용.
- 아이콘 렌더링이 정상적으로 되는지 육안 확인 및 필요시 다크/라이트 모드 대응 아이콘(Template Image) 속성 적용.

## 5. 관련 파일
- `app/src/gui.py`: `QIcon("app/src/ui/tray_icon.png")` 경로 로드 부분
- `build.sh`: PyInstaller 리소스 복사(`--add-data`) 부분 일치 여부 확인
