# 버그: 단축키 설정 시 불필요한 파일 저장 반복 발생

## 1. 발견된 이슈
- **증상**: 설정 창에서 단축키를 조작하거나 창을 이동할 때마다, 로그에 `설정 파일 저장 성공` 메시지가 과도하게 출력됨.
- **원인**: GUI 이벤트 핸들러(`on_gap_changed` 등)가 값이 변경될 때마다 매번 `config_manager.save_config()`를 호출하여 불필요한 디스크 I/O를 유발하고 있음.
- **영향**: 
  - 디스크 I/O 부하 증가.
  - 설정 파일 무결성 위험.
  - 로그 파일 용량 비정상적 증가.

## 2. 향후 작업 (TODO)

- [x] **로그 최적화**: 설정 변경 시에만 로그를 남기도록 수정.
- [x] **불필요한 저장 제거**: `load_config` 호출 시 `save_config`가 무조건 실행되던 문제를 수정하고 메모리 캐시인 `get_config`를 활용하도록 변경.

## 3. 진행 상황 업데이트
- 디바운스 로직(1초)을 GUI 클래스에 도입했으나, UI 초기화 시 불필요한 이벤트 트리거로 인해 반복 저장되는 버그가 잔존함. 추가 디버깅 필요.
- **[완료]** 키보드 이벤트 발생 시마다 설정 파일 로드와 저장이 일어나는 근본 원인(`config_manager.load_config` 내부의 `save_config` 호출)을 식별하고 제거함.
- `get_config()` 캐시 메소드를 도입하여 빈번한 설정 읽기 연산을 메모리 참조로 최적화함. E2E 테스트 업데이트 및 검증 완료.
