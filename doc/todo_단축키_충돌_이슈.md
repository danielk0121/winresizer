# 단축키 충돌 이슈 및 조치 사항 (TODO)

## 1. 발견된 이슈
- **증상**: 크롬 브라우저가 활성화된 상태에서 `Option + Command + Left/Right` 단축키 입력 시, 윈도우 크기 조절 대신 크롬의 **탭 이동** 기능이 실행됨.
- **원인**: macOS 애플리케이션(크롬 등)이 시스템 전역 단축키보다 우선순위가 높은 자체 단축키를 가지고 있어 이벤트를 가로챔(Intercept).
- **영향**: 기본 설정된 단축키가 유명 앱들과 충돌할 경우 앱의 핵심 기능이 작동하지 않는 것으로 오인될 수 있음.

## 2. 임시 조치 (완료)
- **기본 조합 변경**: 충돌 가능성이 낮은 `Control + Option + Command` 조합으로 기본 단축키를 변경함.
  - `Control + Option + Command + Left`: 좌측 절반
  - `Control + Option + Command + Right`: 우측 절반
  - `Control + Option + Command + Up`: 다음 디스플레이 이동
  - `Control + Option + Command + C`: 중앙 고정
- **코드 반영**: `app/src/config_manager.py` 및 `app/src/main.py`에 반영 완료.
