# 버그 리포트: 창 이동 후 포커스 유실 이슈

## 1. 현상 설명
- 활성 창에서 단축키(예: 좌측 절반)를 입력하여 창을 이동/크기 조절한 직후, 동일한 단축키 또는 다른 단축키를 연달아 입력하면 작동하지 않음.
- 창을 마우스로 한 번 클릭하여 포커스를 다시 잡아주면 다시 정상 작동함.
- 사용자는 "포커스가 나갔다"고 느끼며, 연속적인 단축키 입력(스마트 순환 기능 등)이 불가능한 상태임.

## 2. 재현 경로
1. 임의의 창(예: Chrome)을 활성화함.
2. `⌥⌘←` (좌측 절반) 단축키 입력 -> 창이 좌측 절반으로 이동함.
3. 즉시 다시 `⌥⌘←` 입력 -> 창이 1/3 등으로 변해야 하나 아무 반응 없음.
4. 해당 창을 클릭한 후 다시 입력 -> 정상 작동함.

## 3. 원인 분석 (추정)
- `Accessibility API`를 통해 창의 위치/크기를 변경한 후, macOS가 해당 창의 'Focused' 상태를 일시적으로 해제하거나 `NSWorkspace.sharedWorkspace().frontmostApplication()` 결과가 변경될 가능성이 있음.
- **[추가 원인]** 단축키 리스너(`on_press`)에서 단축키 감지 후 `current_keys.clear()`를 수행함. 이로 인해 사용자가 수식 키(Cmd, Alt)를 누른 상태에서 화살표 키만 다시 눌러 '스마트 순환'을 시도할 때, 수식 키가 세트에서 사라져 있어 두 번째 입력이 무시됨.

## 4. 해결 방안
- 창의 위치/크기 변경(`set_window_bounds`) 직후, 해당 앱을 다시 `activateWithOptions_` 등을 이용해 최상위로 올리고 포커스를 명시적으로 부여함.
- `pynput` 리스너에서 단축키 매칭 후 `current_keys.clear()` 대신, 수식 키를 제외한 키들만 제거하거나 수식 키 상태를 유지하도록 로직 개선.
