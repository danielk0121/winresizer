# 오픈소스 분석 기반 윈도우 매니저 기능 확장 제안서

## 1. 개요
본 제안서는 `macpywm`, `PyGetWindow`, `Rectangle`, `ShiftIt` 등 macOS 환경의 주요 윈도우 관리 오픈소스 프로젝트의 README와 소스 구조를 분석하여, 현재 `winresizer` 프로젝트에 도입 가능한 혁신적인 기능들을 제안합니다.

## 2. 오픈소스 벤치마킹 분석 결과

| 프로젝트 | 핵심 특징 | winresizer 도입 제안 |
| :--- | :--- | :--- |
| **Rectangle** | 스냅 영역 및 URL 스킴 지원 | `winresizer://` URL 스킴을 통한 외부 제어 |
| **ShiftIt** | 동일 방향 반복 시 크기 순환 | 1/2 -> 1/3 -> 2/3 순환 로직 (단축키 절약) |
| **macpywm** | 앱별 자동 배치 규칙 (Programmed) | 앱 이름 기반 자동 레이아웃 프로필 기능 |
| **PyGetWindow** | 윈도우 상태(최대/최소) 정밀 제어 | 윈도우 상태값(isMaximized 등) 감지 로직 강화 |

## 3. 세부 기능 보완 제안

### 3.1 동일 단축키 크기 순환 로직 (Smart Cycling)
- **제안 내용:** 사용자가 '왼쪽 절반' 단축키를 한 번 더 누르면 '좌측 1/3', 또 누르면 '좌측 2/3'로 크기가 순환되게 합니다.
- **기대 효과:** 사용자가 외워야 할 단축키 개수를 획기적으로 줄이면서도 정밀한 제어가 가능해집니다.

### 3.2 앱별 자동 레이아웃 규칙 (Auto-Layout Profiles)
- **제안 내용:** `config.json`에 특정 앱(Bundle ID 또는 Name)에 대한 기본 위치를 설정합니다.
  - 예: "Slack"은 실행 시 항상 우측 1/3, "iTerm2"는 좌측 2/3 배치.
- **기대 효과:** 수동 조작 없이도 업무 환경을 즉시 세팅할 수 있는 자동화 환경 구축.

### 3.3 윈도우 제외 리스트 (Ignore Apps)
- **제안 내용:** 게임이나 특정 그래픽 작업 도구 등 단축키가 겹칠 수 있는 앱을 '무시 목록'에 등록합니다.
- **기대 효과:** 다른 전문 소프트웨어와의 충돌을 방지하고 시스템 안정성을 높입니다.

### 3.4 외부 명령 연동 (URL Scheme)
- **제안 내용:** `winresizer://action/left_half`와 같은 URL 스킴을 등록합니다.
- **기대 효과:** Raycast, Alfred, 혹은 커스텀 쉘 스크립트와 연동하여 `winresizer`의 기능을 확장할 수 있습니다.

## 4. 기술적 구현 방안
1. **순환 로직:** `HOTKEY_CONFIG`에 `cycle_group` 속성을 추가하여 상태를 관리합니다.
2. **자동 배치:** `NSWorkspace`의 `didActivateApplication` 이벤트를 감시하여 앱 활성화 시 위치를 보정합니다.
3. **URL 스킴:** `PyQt5`의 `QDesktopServices` 또는 macOS Native `AppleEventManager`를 연동합니다.

## 5. 결론
단순한 좌표 계산기를 넘어, 사용자의 습관을 이해하고 업무 환경을 자동화하는 '지능형 윈도우 매니저'로의 진화를 목표로 합니다. 위 제안된 기능 중 '크기 순환 로직'을 최우선적으로 구현할 것을 제안합니다.
