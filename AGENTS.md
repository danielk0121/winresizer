# 에이전트 작업 지침 (AGENTS.md)

## 기본 원칙
1. **브랜치 관리**: 모든 작업은 `main` 브랜치를 기준으로 수행한다.
2. **설정 격리**: 유저 홈 디렉토리(`~/`)의 전역 설정보다 프로젝트 내부 설정을 우선하며, 전역 설정에 의존하지 않는다.

## 명명 규칙 (Naming Conventions)
1. **폴더 (Folders)**: 항상 **영어**를 사용한다. (예: `app`, `src`, `spec`, `tests`)
2. **소스 코드 파일 (Source Code Files)**: 파이썬 등 실행 코드는 **영어** 파일명을 사용한다. (임포트 호환성 목적)
3. **문서 및 기타 파일 (Documentation)**: 리드미, 기획서 등은 **한국어** 파일명을 사용한다. (예: `리드미.md`, `요구사항명세서.md`)
4. **코드 내부 (In-code)**: 변수명, 함수명, 클래스명, 코드 주석은 모두 **한국어**를 사용한다.

## 기술적 제약
- macOS 환경의 `pyobjc` 및 `pynput` 라이브러리를 사용한다.
- TDD(테스트 주도 개발) 방식을 따르며, `pytest` 또는 `unittest`를 활용한다.
