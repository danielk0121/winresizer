# 에이전트 작업 지침 (AGENTS.md)

## IMPORTANT !!
- 이 사항들이 안 지켜질 경우, 사용자가 험한 욕설을 할 것임 !
- 코드 수정이 끝나고 커밋을 했는데, main 실행 후 에러가 발생하면 절대로 안된다.
- GUI 기능 수정은 반드시 e2e 테스트로 검증해야 됨. 만약 검증이 성공했는데 버그가 있다면, TDD 를 하는 이유가 없음.
- 검증의 가치가 없는 테스트 코드를 검증하는 행위는 자아도취 자기만족 탁상공론 임. 이러한 검증 테스트 코드가 자주 발생하면 안됨.

## 기본 원칙
1. **브랜치 관리**: 모든 작업은 `main` 브랜치를 기준으로 수행한다.
2. **설정 격리**: 유저 홈 디렉토리(`~/`)의 전역 설정보다 프로젝트 내부 설정을 우선하며, 전역 설정에 의존하지 않는다.

## 명명 규칙 (Naming Conventions)
- **소스 파일 및 폴더**: 파일명, 클래스명, 변수명 등 영어를 사용하며, 일반적인 코딩 관례를 사용한다. (예: `app`, `src`, `spec`, `tests`)
- **소스 파일 예외**: 주석은 모두 한국어를 사용하며, 문자열 내용도 가급적 한국어를 사용한다. 로그 기록 메시지 및 앱 내에서 사용되는 문자열 상수는 한국어를 사용한다.
- **문서 및 기타 파일**: 가급적 **한국어**를 사용한다. (파일명, 파일 내용, 예: `리드미.md`, `요구사항명세서.md`)
- **커밋 메세지**: 한국어를 사용한다.

## 기술적 제약
- macOS 환경의 `pyobjc` 및 `pynput` 라이브러리를 사용한다.
- TDD(테스트 주도 개발) 방식을 따르며, `pytest` 또는 `unittest`를 활용한다.
